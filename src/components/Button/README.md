# Button

`Button` is the main action primitive for clicks, links, icon actions, and grouped controls.

## When to use

- Use it for actions that need clear emphasis and consistent sizing across products.
- Switch `variant`, `size`, `icon`, and `iconOnly` instead of creating ad hoc action styles in app code.

## Notes

- `ButtonGroup` lives in the same folder when actions should be visually grouped.
- Keep destructive intent explicit with `danger` instead of reusing neutral variants with custom colors.
