# Actions

`Actions` renders a compact group of primary actions and an optional overflow menu.

## When to use

- Use it in cards, toolbars, and list rows where a small set of related actions should stay aligned.
- Add `contextMenuItems` when the primary row of buttons should stay short and secondary actions belong in an overflow menu.

## Notes

- `actions` is the main required prop and drives the visible buttons.
- The docs app also extracts the prop contract from source so the API table stays aligned with the implementation.
